# class-Account
Код написан на языке Python
Назначение: реализация функционала обычного банковского счёта с ведением истории операций.

Атрибуты экземпляра:

holder (str) — имя владельца счёта;

_balance (float) — текущий баланс (защищённый атрибут);

operations_history (list) — список записей обо всех операциях.

Методы:

__init__(account_holder, balance)

Инициализирует счёт.

Проверяет, что начальный баланс ≥ 0 (иначе — ValueError).

При положительном начальном балансе создаёт запись о первом внесении средств.

_record_operation(...) (внутренний)

Фиксирует операцию в истории.

Параметры: тип операции, сумма, баланс после, статус, флаг использования кредита.

Добавляет запись с текущей датой и временем.

deposit(amount)

Пополняет счёт на указанную сумму.

Требует положительной суммы (иначе — ValueError).

Возвращает True при успехе, False при ошибке.

Записывает операцию в историю с статусом success/fail.

withdraw(amount)

Снимает средства со счёта.

Проверяет положительность суммы и достаточность баланса.

При успехе уменьшает баланс и фиксирует операцию.

При недостатке средств записывает неудачную попытку.

get_balance()

Возвращает текущий баланс счёта.

get_history()

Формирует читаемую строку с историей операций.

Для каждой операции указывает: дату, тип, сумму, баланс после, статус.

Если операций нет — возвращает сообщение об этом.

2. Класс CreditAccount (кредитный счёт)
Наследование: наследует Account, расширяя его функционал для работы с кредитным лимитом.

Дополнительные атрибуты:

credit_limit (float) — максимальный допустимый отрицательный баланс;

_balance может быть отрицательным (в пределах лимита).

Переопределённые/новые методы:

__init__(account_holder, credit_limit, balance)

Инициализирует кредитный счёт.

Проверяет, что баланс ≥ −credit_limit (иначе — ValueError).

Вызывает конструктор родительского класса.

Добавляет поле available_credit в начальную запись (если есть).

_record_operation(...)

Расширяет запись операции полями:

used_credit — использовались ли кредитные средства;

available_credit — доступный кредит после операции.

withdraw(amount)

Позволяет снимать средства, даже если баланс станет отрицательным.

Проверяет, что новый баланс ≥ −credit_limit.

Фиксирует использование кредита (used_credit = True), если баланс уходит в минус.

deposit(amount)

Пополняет счёт с учётом погашения кредитной части.

Определяет, были ли использованы кредитные средства до пополнения.

Обновляет флаг used_credit в записи операции.

get_credit_limit()

Возвращает установленный кредитный лимит.

get_available_credit()

Рассчитывает доступные кредитные средства: баланс + лимит.

Например, при балансе −2000 и лимите 5000 доступный кредит = 3000.

get_history()

Формирует расширенную историю операций с кредитной информацией.

Для каждой операции указывает:

дату и время;

тип операции;

сумму (с пометкой «(кредитные)» при использовании кредита);

баланс после операции;

доступный кредит;

статус операции.
